CS 4241 Final Project: Webchat Nomic

Gareth Solbeck

Summary

Nomic is a game about making rules. Typically, the game starts with a basic ruleset, which describes how rules may be created, and the game evolves based on the choices of the players. Because of this, games can vary widely.

This is a webchat version of Nomic, where the rules are enforced by the server. To make this possible, rules are limited to the scope of messages: each rule can only refer to the content of the messages. Messages that violate any current rules will not be sent.

Note: the domain of this Nomic implementation is so restricted that it isn't really a Nomic game anymore. The game will probably get a more accurate (and interesting) name during development.

Features

  • Send messages in a group chat
    • Messages are limited by existing rules
  • Propose rules to be voted on
    • Require messages to contain/exclude phrases
    • Limit lengths of messages
    • Replace certain phrases with other phrases
  • Propose repeals of existing rules
  • Vote on proposed rules or repeals

Stretch Goals

  • Multiple simultaneous games
  • Add rules which relate to previous messages
  • Add user attributes (i.e: "points") which rules can operate on
  • Write rule language to allow for complex rules
